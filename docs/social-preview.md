# Social Preview Design Brief

> 1280 × 640 px GitHub social preview image.

---

## Title Text

**SkillForge**

## Subtitle Text

*Create better OpenCode skills from vague user requests.*

---

## Layout

| Area | Content |
|------|---------|
| Left 60% | Title + subtitle stacked top-left. Below them, a simple horizontal arrow diagram: **Vague request → Intent → Spec → Audit → Skill** |
| Right 40% | A dark rounded rectangle resembling a code/terminal card. Inside it, show three short lines:
1. Intent Analysis
2. Skill Spec
3. Quality Audit
Below them, show a small file label: `SKILL.md`.
Do not use strikethrough or an X overlay. |

---

## Color / Theme

- Background: dark (#0d1117, GitHub dark default)
- Title: white (#ffffff)
- Subtitle: accent blue (#58a6ff)
- Arrow diagram: muted teal (#3fb950) for arrows, dim white for labels
- Terminal box border: dim gray (#30363d)

---

## Diagram Text

```
Vague request → Intent → Spec → Audit → Skill
```

Use simple Unicode arrows (→) between each step. Each step label should be short (one word or a contraction). The arrows should flow left to right.

---

## What Not to Include

- No code screenshots or real file contents
- No screenshots of agent conversations
- No logos or branding marks besides the title text
- No "beta", "v0.1", or version badges on the preview itself
- No human faces or stock photos
